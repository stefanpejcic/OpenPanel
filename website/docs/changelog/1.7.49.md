## Polish
- Deleting a user now removes all mailbox data for every email account owned by that user.
- cPanel import now supports importing email accounts and PostgreSQL databases.
- Temporary `.cpanel.site` domains are no longer imported from cPanel backups.
- **OpenAdmin > Users > [Single User] > Overview** now displays the user’s locale.
- Optimized the update preferences process in WP Manager to execute a single command for updating WordPress core, themes, and plugins together.
- Optimized the debug preferences update process in WP Manager to run a single command for all options.

### 🐛 Bug Fixes
- Fixed bug on WordPress Autoinstaller that caused the process to stuck on *Writing database logins to wp-config.php* IF table prefix starts with a number.
- Fixed error creating database on WordPress Autoinstall.
- [Can not create ftp user in user panel #896](https://github.com/stefanpejcic/OpenPanel/issues/896)
- Fixed error `list object has no element 1` on *OpenAdmin > Notifications* if message is malformed in the log file.
- Fixed bug 'unrecognized backup format' on cpanel backup import.
- Fixed bug on cpanel backup import: if admin panel is restarted, import process was killed.
- Fixed 500 error on *WP Manager > Scan* when db is not available.
- Fixed error generating auto-login link on WP Manager for installations in subdirectories.
- Fixed bug on OpenPanel startup that caused empty custom.css or custom.js files to be loaded on all pages.
